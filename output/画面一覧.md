# 画面一覧 — Todo 管理システム

## 1. 概要

本書は、**Todo 管理システム** (Mendix App: `TodoManagementApp`) の全 Mendix Page を一覧化したものである。
画面の物理名・所属モジュール・レイアウト・主要アクションを規定し、後続の 04_画面設計書 / 05_画面遷移図 の基礎情報とする。

- **対象範囲:** ユーザが直接操作するすべての Page (Mendix 標準の `Login` ページは除外)。
- **命名規約:** [mx-cd-convention/page-ui](../../mx-cd-convention/parts/page-ui.md) に準拠。Page 物理名 = `{Entity}_{Suffix}` (`_Overview` / `_NewEdit` / `_View` / `_Select` 等)。Home は `Home_{Device}_{Role}`。

## 2. 前提・制約

- **モジュール構成:**
  - `TodoManagement` — Todo / Category / Tag 等の業務ドメイン
  - `TodoShared` — Snippet, 共通 Layout, 共通 Enumeration
- **Layout:** Web は `Responsive_Main` (Atlas UI 3)、モバイル想定がある場合 `Phone_Main` を併設。Popup 系は `Popup_Default`。
- **利用ロール (Module Role):** `EndUser`, `Administrator`。
- **Mendix 要素種別:** 画面設計書 §2.1.1 の許可 widget のみ使用。`Dropdown` / `Reference Selector` 等の deprecated widget は不可。

## 7. 要確認事項

- ※ コメント機能 (`Comment` entity) を独立画面とするか、`Todo_View` 内 Snippet 内蔵か未確定 → 要件確認。
- ※ 添付ファイル (`Attachment`) の最大サイズ・対応拡張子 → Constant `MAX_ATTACHMENT_SIZE_MB` / `ALLOWED_FILE_EXTENSIONS` で定義予定。
- ※ モバイル対応 (S-014, S-015) の必要性 → ステークホルダーへ確認。
- ※ 通知機能 (期限超過・リマインダ) の有無 → 別画面 (`Notification_Overview`) 要否を確認。
